## 📌 What This Demonstrates

- Selecting DOM elements with `querySelector`
- Using a `do...while` loop to run a block at least once before checking the condition
- Clearing previous DOM content with `innerHTML = ""`
- Dynamically creating `<li>` elements with `createElement`
- Appending elements to the DOM with `appendChild`

## 🎯 Key Takeaways

- A `do...while` loop always runs its block at least once, since the condition is checked after the first execution — unlike a `while` loop, which checks the condition first
- Useful for cases where the action must happen at least one time regardless of the initial condition
- Clearing the container (`innerHTML = ""`) before re-rendering still prevents duplicate list items on repeated clicks
