## 📌 What This Demonstrates

- Selecting DOM elements with `querySelector`
- Using a `switch` statement to compare a value against multiple `case`s
- Using `break` to stop execution once a matching case is found
- Handling unmatched values with a `default` case
- Updating the DOM with `textContent`

## 🎯 Key Takeaways

- `switch` is a cleaner alternative to long `if...else if` chains when comparing one value against several fixed options
- Forgetting `break` causes "fall-through" — execution continues into the next case even after a match, which is a common bug
- A `default` case acts as a safety net for values that don't match any defined case
